# Install Jenkins on the Controller Server

## Description

Jenkins runs as a service on a dedicated server known as the **controller** (previously referred to as the **master node**).

The controller is responsible for managing the entire Jenkins environment, including:

- Pipeline configurations
- User management
- Plugin management
- Credentials
- Build scheduling
- Connected agent nodes

In production environments, the Jenkins controller is typically deployed on a dedicated machine. This allows it to efficiently manage hundreds of projects, coordinate distributed builds across multiple agents, and handle thousands of requests generated by Continuous Integration (CI) and Continuous Delivery (CI/CD) workflows.

---

## Steps

### Step 1: Update System Packages

Before installing any software, update the system packages to ensure you have the latest security patches and package versions.

```bash
sudo dnf update -y
```

---

### Step 2: Install Java Development Kit (JDK)

Jenkins is a Java application and requires a Java Development Kit (JDK) to run.

Install OpenJDK 21 using:

```bash
sudo dnf install java-21-openjdk -y
```

---

### Step 3: Add the Jenkins Repository

Jenkins is not included in the default package repositories. Add the official Jenkins repository to your system.

```bash
sudo wget -O /etc/yum.repos.d/jenkins.repo https://pkg.jenkins.io/rpm-stable/jenkins.repo
```

This downloads the official Jenkins repository configuration, allowing your package manager to install Jenkins and receive future updates.

---

### Step 4: Install Jenkins

Now that the repository has been added, install Jenkins.

```bash
sudo yum install jenkins
```

---

### Step 5: Reload systemd

After installation, reload the systemd manager configuration so it recognizes the newly installed Jenkins service.

```bash
sudo systemctl daemon-reload
```

---

### Step 6: Enable the Jenkins Service

Enable Jenkins so it starts automatically whenever the server boots.

```bash
sudo systemctl enable jenkins
```

---

### Step 7: Start the Jenkins Service

Start the Jenkins service.

```bash
sudo systemctl start jenkins
```

---

### Step 8: Verify the Installation

By default, Jenkins runs as a web application on **port 8080**.

At this point, Jenkins is installed and running. The web interface will be used later to complete the initial setup and configure the controller.

Before exposing Jenkins to users, it is recommended to place it behind a reverse proxy such as **NGINX**. This provides cleaner URLs, improved security, SSL termination, and better production readiness.

## Conclusion

You have successfully installed Jenkins as a systemd service on the controller server. The service is enabled to start automatically on boot and is currently running on port **8080**.

The next step is to configure an NGINX reverse proxy before completing the Jenkins web-based setup.
